Frequently Asked Questions

Can I change the chart after it is created?

jsEasyCharts is a clean and simple way to put a chart into your page. There are many other great JavaScript charting solutions out there. jsEasyCharts does not give you an interactive chart, if you want a more dynamic chart then please look at any of the following charting solutions:

  • Google Visualization API, Google's own excellent (but quite complicated) interactive charting solution.
  • Bluff, a free solution by James Coglan that uses the canvas element. It looks very good but has limited documentation.
  • Emprise JavaScript Charts, a paid for solution that uses the canvas element.
  • Raphael JavaScript Library, a beautiful collection of different graphs, again using the canvas element.
  • JS Charts, another paid for solution that uses the canvas element.

The data is too small on the chart / The data is all the same on the chart

Google Charts by default treats all your data as values between 0 and 100. This means that if you have values that are within this range and are very close together then you won't get a very informative chart. Also, if any of the values in your data is over 100, the chart will show it as 100.

To get round this problem Google Charts uses data scaling, jsEasyCharts lets you do this using the "dataScaling" feature in the options object of the constructor.

For example, if you wanted a chart using values that were under 100 and all close to each other, you could do this:

jsEasyCharts.line(
	'line1',
	[48,40,45,51,48,43],
	{
		dataScaling: {
			top: 60,
			bottom: 35
		}
	}
);

Likewise, if you had values in your data that were over 100, you could do this: 

jsEasyCharts.line(
	'line1',
	[148,40,145,51,148,43],
	{
		dataScaling: {
			top: 160,
			bottom: 0
		}
	}
);

I want the chart to work on a secure page

You can use jsEasyCharts on a secure page by adding "https: true" to the options object of the constructor. For example:

jsEasyCharts.line(
	'line1',
	[80,15,45,11,70,55],
	{
		https: true
	}
);

Will give you a secure image to place on your page.

The labelling doesn't work when I have a large JSON object

This is a known issue that will be fixed, but in the mean time you can get around it by declaring your labels in the options object of the constructor.

Labels with non alpha-numeric characters break jsEasyCharts, how can I fix this?

If you want to use a non alpha-numeric character as a label in your chart, you will need to write the character in as its escaped equivalent.

For example, to use a plus sign in a label, you could write %2B instead of +. See www.escapecodes.info for a list of escaped non alpha-numeric characters.
