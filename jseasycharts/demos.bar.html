Bar Charts

Constructor

jsEasyCharts.bar(attachTo, dataSource, opts);
attachTo
Type: DOM element | string
Reference of an element on the page where you want the chart to be put. This can be either a string dictating the id of the element or a reference to the element itself.
dataSource
Type: Array | JSON object | id of a table DOM element
The data that you want to populate the chart with. This can be an array/multi-dimensional array, a JSON object or a reference to a table element.
opts
Type: JSON object
Optional configuration setting.
altText
Type: String
Default: none
The text to be placed in the alt attribute of the image tag.

Example

jsEasyCharts.bar(
	'bar1',
	[60,40,10],
	{
		altText: 'Sales figures for this year'
	}
);
axisLabels
Type: String[]
Default: none
Values to use as the labels for the values of your data. These values go across the Y axis of the chart.

Example

jsEasyCharts.bar(
	'bar1',
	[60,40,10],
	{
		axisLabels: ['high', 'average', 'low']
	}
);
barSpacing
Type: number
Default: none
Specify the spacing between bars on the chart in pixels.

Example

jsEasyCharts.bar(
	'bar1',
	[60,40,10],
	{
		barSpacing: 30,
		barWidth: 40
	}
);
barWidth
Type: number
Default: none
Specify the width of the bars on the chart in pixels.

Example

jsEasyCharts.bar(
	'bar1',
	[60,40,10],
	{
		barSpacing: 30,
		barWidth: 40
	}
);
colour
Type: String
Default: none
The RGB value for the colour you want the lines in the chart to be. To set multiple colours use a comma delimited list of RGB values (e.g. '99cc00,cc9900').

Example

jsEasyCharts.bar(
	'bar1',
	[60,40,10],
	{
		colour: '99cc00,cc9900'
	}
);
dataScaling
Type: JSON OBJECT
Default: none
By default the values that you can enter range from 0 - 100. Google Charts will ignore any value you enter which is less than 0, and cap any values higher than 100 at 100. You can override this behaviour with the use of the dataScaling parameter in the options object. Use the dataScaling option to specify a top and bottom value for your data to be scaled against (e.g. dataScaling: {top: 230, bottom: 120}.

Example

jsEasyCharts.bar(
	'bar1',
	[60,40,10],
	{
		dataScaling: {
			top: 230, 
			bottom: 120
		}
	}
);
grouped
Type: boolean
Default: false
Set to true if your dataSource has multiple tracks of data that you want to group together.

Example

jsEasyCharts.bar(
	'bar1',
	{
		january: {
				"Tom": 10,
				"Jake": 20,
				"Michael": 30
		},
		february: {
				"Tom": 40,
				"Jake": 50,
				"Michael": 60
		},
		march: {
				"Tom": 70,
				"Jake": 80,
				"Michael": 90
		}
	},
	{
		grouped: true
	}
);
https
Type: boolean
Default: false
Set to true if you want an image that has a URL that starts with "https" and won't cause a security alert on a secure page.

Example

jsEasyCharts.bar(
	'bar1',
	[60,40,10],
	{
		https: true
	}
);
labels
Type: String[]
Default: none
Values to use as the labels for your data. These values go across the X axis of the chart.

Example

jsEasyCharts.bar(
	'bar1',
	[20,16,33],
	{
		labels: ['Lib Dem', 'Lab', 'Con']
	}
);
legend
Type: string
Default: none
A comma delimited list of values to display in a legend. The legend will use any colour values passed in via the colour parameter in the options object.

Example

jsEasyCharts.bar(
	'bar1',
	[20,16,33],
	{
		colour: '99cc00,cc9900,00cc99',
		legend: 'Lib Dem, Lab, Con'
	}
);
showAxisLabels
Type: boolean
Default: true
Set to false if your dataSource has data for axisLabels but you do not want them to be shown in the chart.

Example

jsEasyCharts.bar(
	'bar1',
	'tableId8',
	{
		showAxisLabels: true
	}
);
showLabels
Type: boolean
Default: true
Set to false if your dataSource has data for labels but you do not want them to be shown in the chart.

Example

jsEasyCharts.bar(
	'bar1',
	'tableId8',
	{
		showLabels: true
	}
);
size
Type: string
Default: '100x100'
'x' delimited values for the height and width of the chart.

Example

jsEasyCharts.bar(
	'bar1',
	[60,40,10],
	{
		size: '200x150'
	}
);
type
Type: string
Default: 'horizontal'
Set to 'vertical' if you want a vertical bar chart, otherwise leave empty or set to 'horizontal' for a horizontal bar chart.

Example

jsEasyCharts.bar(
	'bar1',
	[60,40,10],
	{
		type: 'vertical'
	}
);

Basic bar chart

You can use the API with very minimal settings. You can create a chart with just a reference to a parent element and an array of values, like so:

jsEasyCharts.bar(
	'bar1',
	[60,40,10]
);

Setting data with JSON, sizing and colouring

As well as defining the data for the chart as an array, you can also pass in a JSON object. If you do this, the labels in the JSON object will be used in the chart. Note that if labels are supplied for the chart, then axis labels will be added also.

A third optional param in the method call is an options object. Here you can set the size and colouring of the chart.

jsEasyCharts.bar(
	'bar4',
	{
		Apricots: 10,
		Walnuts: 20,
		Hazelnut: 30,
		Almond: 40
	},
	{
		colour: 'ff0000',
		size: '110x110'
	}
);

Defining labels and setting data with a HTML table

You can define the data for the chart using a (2 row/column only) table. The options object also allows you to override the labels (from the data source) and set axis labels for the chart.

Conservative 33
Labour 16
Liberal Democrates 20 
jsEasyCharts.bar(
	'bar3',
	'dataBar3',
	{
		type: 'horizontal',
		size: '200x120',
		labels: ['Lib Dem', 'Lab', 'Con'],
		axisLabels: ['0%', '25%', '50%', '75%', '100%']
	}
);

Vertical bar chart

The third param in the method call is an options object. Here you can set the type of chart and also the width and spacing of the bars. Setting the width and spacing of the bars is useful if the labels are wider than the bars.

jsEasyCharts.bar(
	'bar2',
	{
		Apricots: 10,
		Walnuts: 20,
		Hazelnut: 30,
		Almond: 40
	},
	{
		type: 'vertical',
		size: '450x200',
		barWidth: 20,
		barSpacing: 30
	}
);

Data scaling a bar chart

By default the values that you can enter range from 0 - 100. Google Charts will ignore any value you enter which is less than 0, and cap any values higher than 100 at 100. You can override this behaviour with the use of the dataScaling parameter in the options object.

jsEasyCharts.bar(
	'bar5',
	{
		November: 80,
		December: -70,
		January: 120,
		Febuary: -40
	},
	{
		type: 'vertical',
		size: '250x300',
		barWidth: 50,
		barSpacing: 5,
		dataScaling: {
			top: 140, 
			bottom: -80
		},
		axisLabels: ['-80', '-60', '-40', '-20', '0', '20', 
			    '40', '60', '80', '100', '120', '140']
	}
);

Multiple data tracks with a JSON object in a bar chart

You can define more than one track of data by passing in a JSON object with multiple sub JSON objects. You can then set the colour of each track by passing in a list of comma seperated RGB values into the colour param of the options object. By default the bar chart will think multiple tracks are to be rendered together in a single bar...

jsEasyCharts.bar(
	'bar6',
	{
		january: {
			"Tom": 10,
			"Jake": 20,
			"Michael": 30
		},
		february: {
			"Tom": 40,
			"Jake": 50,
			"Michael": 60
		},
		march: {
			"Tom": 70,
			"Jake": 80,
			"Michael": 90
		}
	},
	{
		size: '400x120',
		colour: '99cc00,cc9900,0099cc'
	}
);

... however if you want multiple tracks to appear seperately, you can. Set the param 'grouped' to 'true' in the options parameter. You can also give the chart a legend, so users know what multiple tracks relate to. Add the param "legend" to the options object, give it the value of a comma or pipe seperated list.

jsEasyCharts.bar(
	'bar7',
	{
		january: {
			"Tom": 10,
			"Jake": 20,
			"Michael": 30
		},
		february: {
			"Tom": 40,
			"Jake": 50,
			"Michael": 60
		},
		march: {
			"Tom": 70,
			"Jake": 80,
			"Michael": 90
		}
	},
	{
		type: 'vertical',
		grouped: true,
		size: '350x300',
		colour: '99cc00,cc9900,0099cc',
		legend: 'Tom,Jake,Michael'
	}
);
